## Взаимодействие с контейнером docker

### Задание

1. Компиляция файлов в контейнере
   1. Загрузите образ `cmake.docker` (или скачайте [отсюда](https://hub.docker.com/r/matrim/cmake-examples/)).
   2. Запустите этот образ с ключом `-it`.
   3. Создайте на компьютере файл `hello.c` и наберите в ней код программы, выводящий "hello, world!" на экран.
   4. Скопируйте программу с комьютера в контейнер: 

        > `docker cp путь_к_файлу_на_ компьютере/hello.c имя_контейнера:/home/devuser`, 

        например: 

        >`docker cp c:/work/docker/hello.c frosty_lehmann:/home/devuser`.

   1. В терминале контейнера с помощью команды `cd` перейдите в каталог `/home/devuser`, убедитесь, что в нем есть файл `hello.c` и выполните команду `gcc hello.c`. После успешной компиляции в каталоге должен появиться файл `a.out`. Запустите его с помощью команды `./a.out` (в linux для указания файла в текущем каталоге нужно использовать путь `./`).
   2. Вставьте в файл `hello.c` следующий код:
        
        ```C
        #include "stdio.h"
        main () {
            #if defined(_WIN32)
                printf("hello, windows\n");
            #elif defined(__linux__)
                printf("hello, linux\n");
            #elif defined(__APPLE__)
                printf("hello, Apple\n");
            #elif defined(BSD)
                printf("hello, BSD\n"); 
            #endif
        }
        ```

        Скомпилируйте и запустите его на компьютере под windows и в контейнере под Linux
2. Настройка отображение папки физического компьютера (windows) на каталог виртуальной машины (Linux).
   1. Остановите виртуальную машину, если она запущена.
   2. В VirtualBox откройте настройки для виртуальной машины docker, перейдите в раздел «общие папки», нажмите иконку добавления новой папки. Укажите имя папки на вашем компьютере, например, `d:\docker`, имя папки, например `docker`, можно поставить галочку «Создать постоянную папку».
   3. Запустите виртуальную машину.
   4. В терминале виртуальной машине создайте каталог командой `mkdir имя_папки` (например `mkdir work`). Уточните полный путь к каталогу командой `pwd`.
   5. Смонтируйте созданный каталог на папку на компьютере командой 

        > `sudo mount -t vboxsf -o uid=1000,gid=50 имя_папки полный_путь_к_каталогу`,

        например: 

        > `sudo mount -t vboxsf -o uid=1000,gid=50 docker /home/docker/work`.
        
        При необходимости команду в терминале Linux можно разделить на несколько строк с помощью символа `\` в конце переносимых строк.

   7. Создайте на компьютере в монтируемой (расшаренной) папке файл `hello.txt` и наберите в нем: "hello, docker". На виртуальной машине с помощью команды `cd` перейдите к смонтированному каталогу. С помощью команды `ls` посмотрите его содержимое, убедитесь, что там есть файл `hello.txt`. С помощью команды `more hello.txt` посмотрите содержимое этого файла.
   8. Для автоматического монтирования папок создайте на компьютере файл `init_docker.txt` примерно следующего содержимого:
   
    ```
    mkdir /home/docker/work
    sudo mount -t vboxsf -o uid=1000,gid=50 docker /home/docker/work
    exit
    ```

    Создайте файл `start_docker.bat`:

    ```
    docker-machine start
    docker-machine ssh < init_docker.txt
    ```

    Остановите виртуальную машину и запустите ее командой `start-docker`. Убедитесь, что папки отобразились верно.

3. Настройка отображения каталога виртуальной машины на каталог контейнера
   1. Для отображения каталога виртуальной машины нужно при запуске контейнера указать ключ 
   
        > `-v каталог_виртуального_компьютера:каталог_контейнера`, 
        
        например 
        
        > `docker run -it -v /home/docker/work:/home/devuser cmake`. 
        
        Отобразите в рабочий каталог контейнера каталог виртуальной машины, смонтированный на папку компьютера.
   2. Внесите изменения в файл на компьютере с Windows, убедитесь, что эти изменения произошли и в каталоге контейнера.
   3. Скопируйте любую лабораторную работу из 1-го семестра по программированию
в расшаренную папку, скомпилируйте ее в контейнере и запустите.
