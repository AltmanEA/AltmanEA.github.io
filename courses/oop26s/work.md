# Основные положения

## Тема 

"Веб-приложение" (указывается на титульном листе пояснительной записки).

[Материалы (стандарты оформления, практики разработки и др.)](https://altmanea.ru/doc/)

## Технологический стек 

- TypeScript
- React

Допускается, и даже рекомендуется, подключать дополнительные библиотеки. При этом требуется обоснование для необходимости подключения библиотек, и обоснование выбора этих библиотек (сравнение с аналогами).

## Рассматриваемые вопросы

Реализуется только приложение в браузере. Для сохранения данных можно использовать localStorage. Аутентификация, передача данных по сети, работа с БД НЕ рассматривается.

## План выполнения

В скобках указывается неделя занятий в семестре. Материалы нужно предоставить очно на лабораторном занятии или на занятии КСР.

Расположение недель в календаре можно посмотреть в [КУГ](https://www.omgups.ru/sveden/education/kug/kug_bak.pdf).

При отставании от плана будет снижаться оценка за работу!

- Согласование темы, технического задания (6, 1 контрольная неделя)
- Разработка MVP (11, 2 контрольная неделя)
- Готовый проект (13)
- Первый вариант пояснительной записки (14)
- Защита работы (16)

# Варианты заданий

#### 1. Веб-приложение «Таблицы Шульте» с анализом динамики внимания

##### Описание пользовательского интерфейса

Пользователь видит квадратную таблицу (по умолчанию 5×5) со случайно размещёнными числами, таймер, индикатор текущего числа для поиска и кнопку начала новой попытки. После завершения отображается экран результатов.

##### Действия пользователя

Пользователь должен последовательно нажимать числа в возрастающем порядке (1 → 25) за минимальное время.

##### Обязательные требования

1. Выбор размера таблицы (минимум 3×3 и 5×5).
2. Корректная генерация случайной перестановки чисел.
3. Измерение времени выполнения с точностью до 0.01 сек.
4. Сохранение результатов в localStorage.
5. Экран истории попыток.
6. Визуализация прогресса (график изменения времени).
7. Не менее 10 React-компонентов.
8. Покрытие тестами:

   * генерации таблицы;
   * проверки корректности клика;
   * логики подсчёта времени.

##### Дополнительный функционал (по выбору студента)

* Цветовой режим.
* Обратный порядок поиска.
* Адаптивная сложность.
* Тепловая карта кликов.

#### 2. Веб-приложение «N-back тренажёр рабочей памяти»

##### Описание пользовательского интерфейса

Пользователь видит сетку 3×3. Через равные промежутки времени подсвечивается одна из клеток. Присутствует кнопка «Совпадает», таймер и панель статистики.

##### Действия пользователя

Пользователь должен нажимать кнопку, если текущая позиция совпадает с позицией N шагов назад.

##### Обязательные требования

1. Поддержка режимов 1-back и 2-back.
2. Корректная генерация последовательности стимулов.
3. Подсчёт правильных ответов, ложных срабатываний и пропусков.
4. Настройка скорости предъявления стимулов.
5. Сохранение статистики в localStorage.
6. Покрытие тестами:

   * алгоритма проверки совпадений;
   * генерации последовательности;
   * расчёта точности.

##### Дополнительный функционал

* Режим 3-back.
* Dual N-back (позиция + цвет).
* График прогресса.
* Звуковое сопровождение.

#### 3. Веб-приложение «Stroop-тест»

##### Описание пользовательского интерфейса

Пользователь видит слово, обозначающее цвет, отображённое цветным шрифтом. Ниже располагаются кнопки выбора цвета и таймер.

##### Действия пользователя

Пользователь должен выбрать цвет шрифта, игнорируя текст слова.

##### Обязательные требования

1. Минимум 30 стимулов за одну сессию.
2. Случайная генерация конгруэнтных и неконгруэнтных стимулов.
3. Подсчёт среднего времени реакции, количества ошибок и индекса интерференции.
4. Экран итогового отчёта.
5. Покрытие тестами:

   * генерации стимулов;
   * расчёта итоговых показателей.

##### Дополнительный функционал

* Несколько уровней сложности.
* Визуализация распределения времени реакции.
* Ограничение по времени выполнения.

#### 4. Веб-приложение «Тренажёр реакции (Go/No-Go)»

##### Описание пользовательского интерфейса

В центре экрана появляется визуальный сигнал (например, зелёный или красный круг). Отображается таймер и счётчик попыток.

##### Действия пользователя

Пользователь должен нажимать клавишу (например, пробел) только при появлении целевого сигнала и игнорировать запрещающий сигнал.

##### Обязательные требования

1. Случайная генерация сигналов.
2. Случайная задержка появления сигнала.
3. Подсчёт средней реакции, количества ложных нажатий и пропусков.
4. Минимум 20 попыток за одну сессию.
5. Сохранение результатов.
6. Покрытие тестами логики подсчёта статистики.

##### Дополнительный функционал

* Несколько уровней сложности.
* Различные формы сигналов.
* Анализ стабильности реакции.

#### 5. Веб-приложение «Запоминание последовательностей»

##### Описание пользовательского интерфейса

Пользователь видит поле из кнопок или ячеек, которые последовательно подсвечиваются. После демонстрации последовательности появляется режим ввода.

##### Действия пользователя

Пользователь должен воспроизвести последовательность в правильном порядке.

##### Обязательные требования

1. Автоматическое увеличение длины последовательности.
2. Подсчёт максимальной достигнутой длины.
3. Система уровней сложности.
4. История результатов.
5. Покрытие тестами:

   * генерации последовательности;
   * проверки корректности ответа.

##### Дополнительный функционал

* Ограничение времени на ввод.
* Цветовой режим.
* Звуковая индикация.

#### 6. Веб-приложение «Тренажёр переключения задач»

##### Описание пользовательского интерфейса

Пользователь видит символ (число или букву) и индикатор текущего правила (например, «чётное/нечётное» или «гласная/согласная»).

##### Действия пользователя

Пользователь должен выбрать правильный вариант ответа в соответствии с текущим правилом. Правило периодически меняется.

##### Обязательные требования

1. Автоматическая смена правила по заданному алгоритму.
2. Подсчёт времени реакции при смене правила и без смены.
3. Расчёт показателя когнитивной гибкости.
4. Экран аналитики результатов.
5. Покрытие тестами алгоритмов расчёта показателей.

##### Дополнительный функционал

* Несколько типов правил.
* Усложнение логики выбора.
* Графическое сравнение времени реакции.

#### 7. Платформа из трёх когнитивных мини-тренажёров

##### Описание пользовательского интерфейса

Пользователь видит главное меню с выбором одного из трёх тренажёров. Для каждого тренажёра предусмотрен экран выполнения задания и общий экран прогресса.

##### Действия пользователя

Пользователь выполняет задания в различных тренажёрах и просматривает суммарный прогресс.

##### Обязательные требования

1. Реализация минимум трёх различных тренажёров.
2. Общая система хранения результатов.
3. Единый интерфейс отображения статистики.
4. Использование маршрутизации (React Router).
5. Не менее 12 React-компонентов.
6. Покрытие тестами бизнес-логики.

##### Дополнительный функционал

* Система достижений.
* Элементы геймификации.
* Настройки сложности для каждого тренажёра.
