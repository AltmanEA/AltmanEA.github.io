## Проброс сети и git

### Задание

1. Настройки для контейнера gitbucket
   1. Загрузите образ `gitbucket.docker` (или скачайте [отсюда](https://hub.docker.com/r/gitbucket/gitbucket/)).
   2. Зайдите в настройки виртуальной машины в раздел «Сеть», далее «Адаптер 1 - Проброс портов». В настройках должен быть указать проброс 22 порта, который используется для команды `docker-machine ssh`. Добавьте проброс порта, назовите его, например, `gitbucket`, укажите порт гостя `8080` (порт, на котором мы потом запустим веб-сервер в контейнере) и порт хоста `8080` (порт, на котором будет виден сервер по адресу физического компьютера).
   3. Создайте на компьютере папку для хранения данных контейнера и отобразите ее на папку виртуальной машины (например, на `/home/docker/git`), допишите команды по ее созданию в файл `init-docker.txt` из предыдущей работы.
   4. Запустите контейнер командой 
    
    > docker run -d -p 8080:8080 -v /home/docker/git:/gitbucket gitbucket/gitbucket

    5. С помощью команды `ipconfig` узнайте ip-адрес своего компьютера. Наберите в браузере `ip_адрес:8080` и убедитесь, что работает web-интерфейс GitBucket.
 1. Проверка git
    1. Авторизуйтесь в GitBucket («sign in», логин и пароль -- 'root').
    2. Создайте новый репозиторий, например 'test' (нажмите на иконку с плюсом в правом верхнем углу страницы).
    3. В репозитории создайте файл `hello.txt`, наберите в нем "hello, git!".
    4. В веб-интерфейсе найдите адрес репозитория и скопируйте его в буфер обмена, нажав кнопку `copy to clipboard`.
    5. Запустите образ `cmake` из предыдущей лабораторной работы. В терминале контейнера наберите команду 

    > git clone адрес_из_буфера_обмена

    6. Убедитесь, что в папке, отображающей каталоги контейнера `cmake`, появилась папку с именем репозитория, и она содержит файл `hello.txt` с корректным содержимым.
